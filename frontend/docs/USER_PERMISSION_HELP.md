# 사용자 관리 vs 권한 관리 가이드

## 📌 요약

### 사용자 관리 (`/admin/users`)
- **목적**: 계정 생명주기 및 시스템 기본 역할 관리
- **주요 기능**:
  - 가입 승인/거부
  - 사용자 삭제
  - 활성화/비활성화
  - **시스템 역할 변경** (ORG_ADMIN, OPERATOR, CUSTOMER_ADMIN, CUSTOMER_USER)

### 권한 관리 (`/org/settings/users`)
- **목적**: 커스텀 역할 할당 및 세부 권한 설정
- **주요 기능**:
  - **커스텀 역할 할당** (조직별 맞춤 역할)
  - 접근 범위 조정
  - 개별 권한 설정
  - 권한 오버라이드

## 🎯 권장 워크플로우

1. **사용자 관리**에서 가입 승인 및 시스템 역할 설정
2. 기본 권한으로 충분하면 완료
3. 추가 권한 필요 시 **권한 관리**에서 커스텀 역할 할당
4. 필요시 개별 권한 세부 조정

## 💡 핵심 차이점

| 구분 | 사용자 관리 | 권한 관리 |
|------|------------|----------|
| **관리 대상** | 시스템 기본 역할 | 커스텀 역할 |
| **권한 범위** | 고정된 기본 권한 | 유연한 맞춤 권한 |
| **사용 시점** | 계정 생성/삭제 시 | 세밀한 권한 조정 시 |
| **변경 빈도** | 낮음 | 높음 |
